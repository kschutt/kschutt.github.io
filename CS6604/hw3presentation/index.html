Twitter API

Streaming API

Allows for the user to specify the minimum bounding box for a specific location in the following format MinLongitude,MinLatitude,MaxLongitude,MaxLatitude

No limitations since you just open a web socket and ingest the data.

Places API

More robust and can search on a polygon with multiple lat-lon pairs. However, this is only obtaining "places" information from twitter and not tweets.

Subject to Twitter's Rate Limiting found here

Twitter Streaming API Gotcha's

Must verify that the received coordinates are correct and within the provided bounding box 


	private bool ValidLatitude(double latitude)
        {
            return latitude >= -90 && latitude <= 90;
        }
        
        private bool ValidLongitude(double longitude)
        {
            return longitude >= -180 && longitude <= 180;
        }
        
        private bool IsValidInBoundingBox(Coordinate coordinate)
        {
            return coordinate.Latitude <= NortheastLatitude 
            	&& coordinate.Latitude >= SouthwestLatitude
                && coordinate.Longitude <= NortheastLongitude 
                && coordinate.Longitude >= SouthwestLongitude;
        }

Twitter Streaming API with Polygon

After the coordinates are verified, we can check against the bounds of a polygon (i.e. Point in Polygon) 


KmlFile file = KmlFile.Load(_kmlPath);
polygonCoords = new List<PointF>();
foreach (var poly in file.Root.Flatten().OfType<Polygon>())
{
    foreach (var coord in poly.OuterBoundary.LinearRing.Coordinates)
    {
        polygonCoords.Add(new PointF((float)coord.Longitude, (float)coord.Latitude));
    }
}

private bool IsPointInPolygon(PointF point)
{
    PointF[] polygon = polygonCoords.ToArray();
    bool isInside = false;
    for (int i = 0, j = polygon.Length - 1; i < polygon.Length; j = i++)
    {
        if (((polygon[i].Y > point.Y) != (polygon[j].Y > point.Y)) &&
        (point.X < (polygon[j].X - polygon[i].X) * (point.Y - polygon[i].Y) / 
        (polygon[j].Y - polygon[i].Y) + polygon[i].X))
        {
            isInside = !isInside;
        }
    }
    return isInside;
}

Foursquare Search API

More details here

Users can utilize one of the following parameters when querying:

  • ll - Location
  • near - A place name, e.g., Washington, DC
  • radius - Used with ll and in meters
  • sw and ne - A bounding quadrangle with an area < 10,000 km

Gotcha's

There are a few issues with using the data from these services.

  • Inaccuracies are everywhere
  • Sometimes the locations are not within the region specified
  • Generally requires some post processing

Example

Example: Geospatial SQL View

Query on a non-spatial attribute 

							
SELECT      
  st_FlipCoordinates(st_intersection(a.the_geom, b.the_geom))
FROM
  "opengeo"."VBMP_RCL_SHP" as a,
  "opengeo"."VBMP_RCL_SHP" as b
WHERE
  a."SN_FULL" % '%primary_name%'
  and b."SN_FULL" % '%secondary_name%'
  AND ST_Touches(a.the_geom, b.the_geom)
  AND a.fid != b.fid
GROUP BY
  st_FlipCoordinates(st_intersection(a.the_geom, b.the_geom))

Example: Geospatial SQL View

Query on a spatial attribute 

							
SELECT
  a.fid, a."SN_FULL", a."VDOT_RNUM", st_flipcoordinates(st_buffer(a.the_geom,0.000000000001))  as the_geom
FROM
  "opengeo"."VBMP_RCL_SHP" as a
WHERE
  st_intersects(a.the_geom, st_buffer(st_pointfromtext('POINT(%lat_lon%)',4269), 0.0005))
ORDER BY
  st_distance(st_pointfromtext('POINT(%lat_lon%)',4269), a.the_geom)

Note: the use of a really small st_buffer is required in this case because of a weird processing error with st_flipcoordinates
